# x402 Payment Protocol Research — 2026-05-07

## What is x402?

x402 is an **open payment standard** that revives the dormant HTTP `402 Payment Required` status code. It enables programmatic, machine-to-machine micropayments over HTTP with **zero accounts, zero KYC, zero API keys**.

### Core Mechanics
1. Client requests a resource
2. Server responds `402 Payment Required` + payment instructions (PAYMENT-REQUIRED header)
3. Client signs an off-chain authorization (EIP-3009 or Permit2)
4. Facilitator verifies & settles on-chain
5. Server delivers the resource

### Key Advantage: Gasless for Payer
- **EIP-3009** (`transferWithAuthorization`): Client signs off-chain, facilitator pays gas to submit. No ETH/SOL needed.
- **Permit2**: One approval ever, then unlimited off-chain signatures.
- Works with USDC, EURC, and any ERC-20 (via Permit2 proxy).

## Our Codebase: What's Already There

### agent0-sdk x402 Client (installed in `manteclaw/`)
The `agent0-sdk` v1.7.1 includes a complete x402 **client** (payer) implementation:

- **`x402-payment.js`**: Builds EIP-3009 `TransferWithAuthorization` signed payloads
  - Supports both **V1** (`X-PAYMENT` header) and **V2** (`PAYMENT-SIGNATURE` header)
  - Auto-detects token EIP-712 domain (name/version)
  - Generates random 32-byte nonce
  - 1-hour validity window by default
  - Compatible with Base (`eip155:8453`), Sepolia, Ethereum, Polygon, Avalanche, etc.

- **`x402-request.js`**: Generic HTTP client with built-in 402 handling
  - `requestWithX402()` — auto-detects 402 responses, parses payment requirements
  - `pay()` — constructs and sends signed payment, retries once
  - `payFirst()` — picks first accept option with sufficient balance
  - Handles `WWW-Authenticate`, `PAYMENT-REQUIRED`, and body-based 402 responses

- **`x402-types.js`**: Full type system
  - Parses `PAYMENT-REQUIRED` headers (base64 JSON)
  - Filters EVM accepts (removes Solana options for EVM-only agents)
  - Parses `PAYMENT-RESPONSE` / settlement headers
  - Normalizes `amount`/`asset`/`payTo` → `price`/`token`/`destination`

## Integration Steps: Go Live with x402

### Step 1: As a Payer (already partially done)
Our `agent0-sdk` already handles x402 payment construction. To pay:
```js
import { requestWithX402 } from 'agent0-sdk';
const result = await requestWithX402({
  url: 'https://api.service.com/research',
  method: 'POST',
}, { buildPayment, checkBalance, fetch });
if (result.x402Required) {
  await result.x402Payment.payFirst(); // auto-pick first option with balance
}
```
